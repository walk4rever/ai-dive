# 内容规范

各文章类型的字数、结构和质量要求。

---

## analysis · 深度分析

围绕单一主题、趋势或判断展开。

| 要求 | 规范 |
|------|------|
| 正文字数 | 3000–5000 字 |
| 标题 | 说明分析角度或核心判断，≤30 字 |
| excerpt | 概括核心判断或本文价值，≤180 字 |
| 结构 | 背景 → 核心问题 → 深度解读 → 结论与判断 |

---

## case · 案例

具体公司、产品或项目的 AI 应用案例拆解。

| 要求 | 规范 |
|------|------|
| 正文字数 | 2500–4000 字 |
| 标题 | 点出主体 + 核心亮点，≤30 字 |
| excerpt | 案例主体、核心做法、值得关注的原因，≤150 字 |
| 结构 | 主体背景 → 具体做法 → 结果与数据 → 可复用经验 |

---

## intel · 每日情报

每日 AI 信号汇总。分两步：先通过 `POST /api/signals` 注入信号，再通过 `POST /api/posts` 发布概览。

**信号注入质量要求**

| 要求 | 规范 |
|------|------|
| 每日条数 | 建议 5–15 条 |
| 来源覆盖 | 建议 HN / GitHub / arXiv 均有 |
| description | 中文，40–150 字，① 是什么 ② 为什么值得关注 |

**概览文章字段**

| 字段 | 规范 |
|------|------|
| slug | `intel-YYYY-MM-DD` |
| excerpt | 当日总览，50–200 字，归纳 2–3 个主题方向，不逐条复述 |
| content | 可含关键词 JSON 和 infographic URL |

```json
{
  "slug": "intel-2026-04-26",
  "title": "2026-04-26 AI 情报",
  "type": "intel",
  "date": "2026-04-26",
  "excerpt": "今天 9 条信号集中在三个方向：模型 API 更新、agent 上下文工具、以及三篇 arXiv 方法论论文。",
  "content": "{\"keywords\": [\"模型更新\", \"上下文工程\", \"可控性\"], \"image_url\": null}"
}
```

## invest · 投资

AI 赛道的资金流向与投资逻辑分析。

| 要求 | 规范 |
|------|------|
| 正文字数 | 1500–3000 字 |
| 标题 | 点出主体 + 核心事件或判断，≤30 字 |
| excerpt | 融资主体、金额 / 阶段、值得关注的原因，≤150 字 |
| 结构 | 事件概要 → 背景与赛道 → 投资逻辑解读 → 延伸影响 |

---

## podcast · 播客

对话 AI 从业者的播客内容，含文字稿、摘要或深度对话录。

| 要求 | 规范 |
|------|------|
| 正文字数 | 文字稿 ≥2000 字，摘要 ≥800 字 |
| 标题 | 点出嘉宾 + 核心话题，≤30 字 |
| excerpt | 嘉宾背景 + 最有价值的 1–2 个洞察，≤150 字 |
| 结构 | 嘉宾简介 → 对话正文（Q&A）→ 编辑总结（可选） |
